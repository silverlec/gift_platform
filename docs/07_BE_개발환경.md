### 3.2 MySQL 데이터베이스 초기화

MySQL 클라이언트 또는 MySQL Workbench에서 실행:

```sql
-- 1. 데이터베이스 생성
CREATE DATABASE giftpick_db
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- 2. 전용 사용자 생성 (선택 사항, 보안 권장)
CREATE USER 'giftpick_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON giftpick_db.* TO 'giftpick_user'@'localhost';
FLUSH PRIVILEGES;

-- 3. 데이터베이스 선택 확인
USE giftpick_db;
SHOW TABLES; -- 처음에는 빈 테이블 (JPA가 자동 생성)
```

### 3.3 application.yml 설정

`src/main/resources/application.yml` 파일을 편집합니다:

```yaml
server:
  port: 8080  # 백엔드 서버 포트

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/giftpick_db?useSSL=false&serverTimezone=Asia/Seoul&characterEncoding=UTF-8
    username: root           # MySQL 사용자명
    password: your_password  # ⚠️ 본인 MySQL 비밀번호로 변경
    driver-class-name: com.mysql.cj.jdbc.Driver

  jpa:
    hibernate:
      ddl-auto: update       # 개발: update (자동 스키마 생성/수정), 운영: validate
    show-sql: true           # SQL 로그 출력 (개발용)
    properties:
      hibernate:
        format_sql: true
        dialect: org.hibernate.dialect.MySQL8Dialect

# JWT 설정
jwt:
  secret: your-256-bit-secret-key-here-minimum-32-characters  # ⚠️ 안전한 시크릿 키로 변경
  expiration: 86400000  # 토큰 유효시간 (밀리초): 24시간

# Swagger UI 설정
springdoc:
  api-docs:
    path: /api-docs
  swagger-ui:
    path: /swagger-ui.html
```

> **보안 주의**: `jwt.secret`은 최소 32자 이상의 랜덤 문자열을 사용하세요.  
> 운영 환경에서는 환경 변수로 관리하세요: `${JWT_SECRET}`

## 5. 레이어드 아키텍처

```
HTTP Request
     ↓
[Controller Layer]      - HTTP 요청 수신, DTO 변환, 응답 반환
     ↓
[Service Layer]         - 비즈니스 로직, 트랜잭션 관리
     ↓
[Repository Layer]      - JPA를 통한 데이터베이스 조회/저장
     ↓
[Database (MySQL)]
```

**패키지 컨벤션:**
```
com.giftpick.domain.{도메인명}/
├── {Entity}.java              # JPA 엔티티
├── {Entity}Repository.java    # Spring Data JPA 인터페이스
├── {Entity}Service.java       # 비즈니스 로직
├── {Entity}Controller.java    # REST 컨트롤러
└── dto/
    ├── {Entity}Request.java   # 요청 DTO
    └── {Entity}Response.java  # 응답 DTO
```

## 6. 보안 설정 (Spring Security)

### 인증 흐름

```
1. POST /api/auth/login 요청
2. UserService가 이메일/비밀번호 검증 (BCrypt 비교)
3. 검증 성공 → JwtTokenProvider가 JWT 생성 (HS256)
4. JWT를 응답 body에 포함
5. 이후 모든 보호된 API 요청에 Authorization 헤더 포함
6. JwtAuthenticationFilter가 토큰 검증
7. SecurityContext에 Authentication 저장
```

### 접근 권한 설정 (SecurityConfig.java)

```java
// 공개 경로 (인증 불필요)
PermitAll:
  - POST /api/auth/register
  - POST /api/auth/login
  - GET  /api/invite/**

// GIVER 전용
hasRole("GIVER"):
  - POST   /api/events
  - GET    /api/events
  - PUT    /api/events/**
  - DELETE /api/events/**
  - POST   /api/events/**/send
  - PATCH  /api/events/**/complete
  - POST   /api/events/**/options
  - DELETE /api/events/**/options/**

// RECIPIENT 전용
hasRole("RECIPIENT"):
  - POST /api/events/**/select

// Giver or Recipient (동적 검증)
authenticated:
  - GET /api/events/{id}
  - GET /api/events/**/selection
```

## 8. 성능 최적화 가이드

### N+1 문제 방지

GiftEvent 조회 시 연관 GiftOption과 GiftSelection을 함께 로드:

```java
// GiftEventRepository.java - Fetch Join 적용
@Query("SELECT DISTINCT e FROM GiftEvent e " +
       "LEFT JOIN FETCH e.options " +
       "LEFT JOIN FETCH e.selection " +
       "WHERE e.id = :id")
Optional<GiftEvent> findByIdWithDetails(@Param("id") Long id);
```

### 인덱스 활용
- `giver_id` 인덱스: Giver 이벤트 목록 조회 최적화
- `recipient_id` 인덱스: Recipient 초대 이벤트 조회 최적화
- `event_id` UNIQUE 인덱스 on gift_selections: 중복 선택 방지

## 11. 트러블슈팅

| 문제 | 원인 | 해결 방법 |
|------|------|----------|
| `Access denied for user 'root'@'localhost'` | MySQL 비밀번호 불일치 | `application.yml`의 `password` 수정 |
| `Unknown database 'giftpick_db'` | DB 미생성 | MySQL에서 `CREATE DATABASE giftpick_db` 실행 |
| `Port 8080 already in use` | 포트 충돌 | `application.yml`에서 `server.port: 8090`으로 변경 |
| `JWT expired` | 토큰 만료 | 재로그인 후 새 토큰 사용 |
| Lombok 어노테이션 미동작 | IDE Annotation Processor 미활성화 | IntelliJ: `Settings → Build → Annotation Processors → Enable` |
| `ClassNotFoundException: com.mysql.cj.jdbc.Driver` | MySQL Connector 미포함 | `./gradlew build` 재실행 |
