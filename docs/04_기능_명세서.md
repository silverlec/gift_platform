## 2. 인증 기능 상세

### FS-01: 회원가입

**화면**: `/register`

**입력 항목:**

| 필드 | 타입 | 필수 | 유효성 규칙 |
|------|------|------|------------|
| 사용자명 | TEXT | ✅ | 2자 이상 50자 이하 |
| 이메일 | EMAIL | ✅ | 이메일 형식, 중복 불가 |
| 비밀번호 | PASSWORD | ✅ | 8자 이상, 영문+숫자 조합 권장 |
| 비밀번호 확인 | PASSWORD | ✅ | 비밀번호 일치 여부 검증 |
| 역할 선택 | RADIO | ✅ | GIVER 또는 RECIPIENT 중 1개 선택 |

**처리 흐름:**
```
1. 사용자가 폼 입력 후 [회원가입] 버튼 클릭
2. 클라이언트 측 유효성 검사 (빈칸, 이메일 형식, 비밀번호 일치)
3. POST /api/auth/register 호출
4. [성공] JWT 자동 발급 후 역할에 따라 대시보드로 이동
5. [실패-중복 이메일] "이미 사용 중인 이메일입니다" 에러 표시
6. [실패-유효성 오류] 해당 필드에 인라인 에러 표시
```

**상태 변화**: 없음 → 사용자 계정 생성됨

---

### FS-02: 로그인

**화면**: `/login`

**입력 항목:**

| 필드 | 타입 | 필수 | 유효성 규칙 |
|------|------|------|------------|
| 이메일 | EMAIL | ✅ | 이메일 형식 |
| 비밀번호 | PASSWORD | ✅ | 필수 입력 |

**처리 흐름:**
```
1. 사용자가 이메일/비밀번호 입력 후 [로그인] 버튼 클릭
2. POST /api/auth/login 호출
3. [성공] JWT를 localStorage에 저장, 역할별 대시보드로 이동
   - GIVER → /giver/dashboard
   - RECIPIENT → /events (초대된 이벤트 목록)
4. [실패] "이메일 또는 비밀번호가 올바르지 않습니다" 오류 표시
```

**데모 기능**: 데모 계정 자동 로그인 버튼 제공 (giver@example.com, minsu@example.com)

## 3. Giver 기능 상세

### FS-05: Giver 대시보드

**화면**: `/giver/dashboard`  
**접근 권한**: GIVER 역할 (JWT 필요)

**화면 구성:**

| 컴포넌트 | 내용 | API |
|----------|------|-----|
| 환영 배너 | 로그인한 Giver 이름 표시 | GET /api/auth/me |
| 통계 카드 | 전체 / 진행중 / 완료된 이벤트 수 | GET /api/events |
| 최근 이벤트 목록 | 최신 5개 이벤트 카드 | GET /api/events |
| 빠른 액션 버튼 | [새 이벤트 만들기] | - |

---

### FS-06: 이벤트 생성

**화면**: `/giver/events/new`  
**접근 권한**: GIVER 역할

**입력 항목:**

| 필드 | 타입 | 필수 | 유효성 규칙 |
|------|------|------|------------|
| 이벤트명 | TEXT | ✅ | 2자 이상 200자 이하 |
| 이벤트 설명 | TEXTAREA | ❌ | 최대 1000자 |
| 수혜자 이메일 | EMAIL | ✅ | 시스템에 등록된 RECIPIENT 계정 |
| 시작일 | DATE | ✅ | 오늘 이후 날짜 |
| 종료일 | DATE | ✅ | 시작일 이후 날짜 |
| 초대 메시지 | TEXTAREA | ❌ | 최대 500자 |

**처리 흐름:**
```
1. Giver가 이벤트 정보 입력 후 [이벤트 생성] 버튼 클릭
2. POST /api/events 호출
3. [성공] 생성된 이벤트 상세 페이지로 이동 (선물 옵션 등록 안내)
4. [실패-수혜자 미존재] "등록되지 않은 이메일입니다" 오류 표시
5. [실패-날짜 오류] 날짜 필드에 인라인 에러 표시
```

**상태 변화**: 새 GiftEvent 생성 (status: CREATED)

---

### FS-11: 수혜자 초대 발송

**화면**: `/giver/events/:id` (이벤트 상세 내 버튼)  
**접근 권한**: 해당 이벤트의 Giver

**처리 흐름:**
```
1. Giver가 이벤트 상세 페이지에서 [초대 발송] 버튼 클릭
2. 조건 확인: 선물 옵션이 1개 이상 등록되어 있어야 함
3. POST /api/events/{id}/send 호출
4. [성공] 이벤트 상태 CREATED → SENT
5. 초대 링크 생성 및 표시 (/invite/{eventId})
6. [클립보드 복사] 버튼 제공으로 링크 손쉽게 공유
```

**상태 변화**: GiftEvent.status = CREATED → SENT

---

### FS-12: 선물 옵션 추가

**화면**: `/giver/events/:id` (모달 또는 인라인 폼)  
**접근 권한**: 해당 이벤트의 Giver

**입력 항목:**

| 필드 | 타입 | 필수 | 유효성 규칙 |
|------|------|------|------------|
| 선물명 | TEXT | ✅ | 2자 이상 200자 이하 |
| 선물 설명 | TEXTAREA | ❌ | 최대 500자 |
| 이미지 URL | URL | ❌ | 유효한 URL 형식 |
| 포인트 | NUMBER | ✅ | 0 이상의 정수 |

**처리 흐름:**
```
1. [선물 옵션 추가] 버튼 클릭 → 입력 폼 표시 (모달)
2. 정보 입력 후 [추가] 버튼 클릭
3. POST /api/events/{eventId}/options 호출
4. [성공] 옵션 목록에 새 항목 실시간 추가
5. [실패] 에러 메시지 인라인 표시
```

### FS-14: 구매 완료 처리

**화면**: `/giver/events/:id`  
**접근 권한**: 해당 이벤트의 Giver

**처리 흐름:**
```
1. 이벤트 상태가 SELECTED인 경우에만 [구매 완료] 버튼 활성화
2. 버튼 클릭 → 확인 모달 표시 "선물 전달을 완료했나요?"
3. 확인 시 PATCH /api/events/{id}/complete 호출
4. [성공] 이벤트 상태 SELECTED → COMPLETED
5. 이벤트 카드에 완료 배지 표시
```

**상태 변화**: GiftEvent.status = SELECTED → COMPLETED

## 4. Recipient 기능 상세

### FS-15: 초대장 확인

**화면**: `/invite/:eventId`  
**접근 권한**: 인증 불필요 (PermitAll)

**화면 구성:**

| 구성 요소 | 내용 |
|----------|------|
| 이벤트 헤더 | 이벤트명, Giver 이름 |
| 초대 메시지 | Giver가 작성한 초대 메시지 |
| 이벤트 기간 | 시작일 ~ 종료일, 남은 일수 |
| 액션 버튼 | [선물 선택하러 가기] 버튼 (로그인 필요) |

**처리 흐름:**
```
1. Recipient가 초대 링크(/invite/:eventId) 클릭
2. GET /api/invite/{eventId} 호출 (비인증 허용)
3. 이벤트 정보 화면 표시
4. [선물 선택하러 가기] 클릭 → 로그인되어 있으면 선택 페이지로 이동
5. 로그인 안 된 경우 → 로그인 페이지로 이동 후 리다이렉트
```

---

### FS-16: 선물 선택

**화면**: `/events/:eventId/select`  
**접근 권한**: 해당 이벤트의 Recipient (RECIPIENT 역할 + JWT)

**화면 구성:**

| 구성 요소 | 내용 |
|----------|------|
| 이벤트 정보 | 이벤트명, Giver 이름, 남은 선택 기간 |
| 선물 옵션 목록 | 카드 그리드 형태 (이미지, 이름, 설명, 포인트) |
| 선택 상태 | 선택된 옵션 시각적 강조 (테두리 하이라이트, 체크 아이콘) |
| 선택 완료 버튼 | 옵션 선택 후 활성화 |

**처리 흐름:**
```
1. 선물 옵션 목록 화면 진입
2. GET /api/events/{eventId}/options 로 옵션 목록 로드
3. Recipient가 원하는 옵션 카드 클릭 → 선택 상태로 전환 (시각적 강조)
4. [선택 완료] 버튼 클릭 → 확인 모달 "이 선물로 선택할까요?"
5. 확인 시 POST /api/events/{eventId}/select 호출 ({selectedOptionId})
6. [성공] 선택 완료 페이지(/events/:eventId/selected)로 이동
7. [실패-이미 선택됨] "이미 선택이 완료된 이벤트입니다" 안내
8. [실패-기간 만료] "선물 선택 기간이 종료되었습니다" 안내
```

**상태 변화**: GiftSelection 생성, GiftEvent.status = SENT → SELECTED

**UX 규칙:**
- 옵션은 한 번에 1개만 선택 가능
- 선택 완료 전까지 자유롭게 옵션 변경 가능
- 선택 확정 후에는 변경 불가

**화면**: `/events/:eventId/selected`

## 5. 공통 기능

### 이벤트 상태 배지 (Status Badge)

| 상태 | 배지 색상 | 표시 텍스트 |
|------|---------|------------|
| CREATED | 회색 | 작성 중 |
| SENT | 파란색 | 초대 발송됨 |
| SELECTED | 주황색 | 선택 완료 (구매 대기) |
| COMPLETED | 초록색 | 완료 |

### 에러 처리 공통 규칙

| 상황 | 처리 방법 |
|------|----------|
| 네트워크 오류 | 토스트 메시지: "잠시 후 다시 시도해주세요" |
| 인증 만료 (401) | 로그인 페이지로 자동 리다이렉트 |
| 권한 없음 (403) | "접근 권한이 없습니다" 안내 페이지 표시 |
| 리소스 없음 (404) | "존재하지 않는 페이지입니다" 안내 |
| 서버 오류 (500) | "서버 오류가 발생했습니다" 토스트 메시지 |

### 로딩 상태 처리
- API 호출 중: 스켈레톤 UI 또는 스피너 표시
- 버튼 중복 클릭 방지: API 호출 중 버튼 비활성화
