## 3. 로컬 개발환경 설정

### 3.1 사전 요구사항

```bash
# Node.js 버전 확인 (20.x LTS 이상 필요)
node --version  # v20.x.x 이상

# npm 버전 확인
npm --version   # 10.x 이상
```

Node.js 설치: https://nodejs.org/ko (LTS 버전 권장)

### 3.2 프로젝트 설치

```bash
# 프론트엔드 디렉토리로 이동
cd gift-platform-frontend

# 의존성 설치
npm install
```

### 3.3 개발 서버 실행

```bash
# Vite 개발 서버 실행 (기본 포트: 5173)
npm run dev
```

서버 실행 후 브라우저에서 접속:
```
http://localhost:5173
```

프론트엔드는 기본적으로 `http://localhost:8080` 백엔드 서버와 통신합니다.

> **중요**: 프론트엔드 실행 전 백엔드 서버가 먼저 실행되어 있어야 합니다.

## 5. 핵심 아키텍처

### 5.1 API 통신 (src/utils/api.ts)

모든 API 요청은 `api.ts`의 래퍼 함수를 통해 처리됩니다.

```typescript
// JWT 자동 주입 구조
const apiFetch = async (url: string, options?: RequestInit) => {
  const token = localStorage.getItem('token');
  const headers = {
    'Content-Type': 'application/json',
    ...(token ? { Authorization: `Bearer ${token}` } : {}),
    ...options?.headers,
  };
  
  const response = await fetch(`http://localhost:8080${url}`, {
    ...options,
    headers,
  });
  
  return response.json();
};
```

### 5.2 전역 상태 관리 (src/context/AppContext.tsx)

React Context API를 사용하여 전역 인증 상태를 관리합니다.

```typescript
interface AppContextType {
  user: User | null;           // 로그인된 사용자 정보
  token: string | null;        // JWT 토큰
  login: (userData, token) => void;  // 로그인 처리
  logout: () => void;          // 로그아웃 처리
}
```

**localStorage 활용:**
- `token`: JWT 액세스 토큰 저장
- `user`: 사용자 정보 캐싱 (새로고침 대응)

### 5.3 라우팅 구조 (src/App.tsx)

```typescript
// React Router v7 기반 라우팅
<Routes>
  <Route path="/"                           element={<LandingPage />} />
  <Route path="/login"                      element={<LoginPage />} />
  <Route path="/register"                   element={<LoginPage />} />
  
  {/* Giver 전용 (GIVER 역할 필요) */}
  <Route path="/giver/dashboard"            element={<ProtectedRoute role="GIVER"><Dashboard /></ProtectedRoute>} />
  <Route path="/giver/events/new"           element={<ProtectedRoute role="GIVER"><EventForm /></ProtectedRoute>} />
  <Route path="/giver/events/:id"           element={<ProtectedRoute role="GIVER"><EventDetail /></ProtectedRoute>} />
  
  {/* Recipient 전용 */}
  <Route path="/invite/:eventId"            element={<InviteLanding />} />  {/* 비인증 허용 */}
  <Route path="/events/:eventId/select"     element={<ProtectedRoute role="RECIPIENT"><GiftSelect /></ProtectedRoute>} />
  <Route path="/events/:eventId/selected"   element={<ProtectedRoute role="RECIPIENT"><GiftSelected /></ProtectedRoute>} />
</Routes>
```

## 9. 빌드 및 배포


### 배포 옵션

| 플랫폼 | 방법 |
|--------|------|
| **Nginx** | `dist/` 폴더를 정적 파일 서버로 서빙 |

**Nginx 설정 예시 (SPA 라우팅 처리):**
```nginx
server {
  listen 80;
  root /var/www/giftpick;
  index index.html;
  
  location / {
    try_files $uri $uri/ /index.html;  # SPA 라우팅 폴백
  }
}
```

## 10. 트러블슈팅

| 문제 | 원인 | 해결 방법 |
|------|------|----------|
| CORS 오류 | 백엔드 CORS 미설정 | 백엔드 `CorsConfig.java` 확인, 프론트 Origin 허용 여부 확인 |
| 로그인 후 빈 화면 | JWT 저장 실패 | 브라우저 localStorage 접근 허용 여부 확인 |
| API 401 오류 | 토큰 만료 | 다시 로그인하여 새 토큰 발급 |
| 포트 충돌 | 5173 포트 사용 중 | `npm run dev -- --port 3000` 으로 포트 변경 |
| 패키지 충돌 | node_modules 손상 | `rm -rf node_modules && npm install` |
